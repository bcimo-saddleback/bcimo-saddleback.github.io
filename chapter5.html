Chapter5: Publishing Your Website

Introduction

Publishing your website is the final step in web development, allowing you to share your work with the world. In this chapter, you will learn about traditional and modern methods of website publishing, including FTP, cloud hosting services, and GitHub Pages. You will also get hands-on experience by publishing a simple website using GitHub Pages and verifying its functionality.

Brief Overview of FTP and Secure FTP (SFTP)

What is FTP?

File Transfer Protocol (FTP) is a standard network protocol used to transfer files from a local computer to a web server. It has been a fundamental tool for website publishing since the early days of the internet. However, FTP lacks security features, making it vulnerable to unauthorized access.

Secure FTP (SFTP)

SFTP (Secure File Transfer Protocol) is a more secure version of FTP that encrypts file transfers using Secure Shell (SSH). This prevents unauthorized interception of data, making SFTP the preferred choice for transferring website files securely.

Using FTP/SFTP to Upload a Website

To upload a website using FTP or SFTP, follow these steps:

  1. Choose an FTP Client: Software like FileZilla or Cyberduck can connect to your web server.

  2. Connect to Your Web Host: Enter your hosting provider’s FTP/SFTP credentials.

  3. Upload Your Files: Drag and drop your website files into the public directory (usually named public_html or www).

  4. Verify Your Website: Open a browser and enter your domain name to check if your site is live.

Modern Methods for Publishing a Website

While FTP/SFTP remains relevant, modern web development offers more streamlined and automated publishing methods:

1. Cloud Hosting Services

Cloud hosting services like AWS, Google Cloud, and Netlify simplify website deployment by offering automated tools, content delivery networks (CDNs), and scalable storage solutions. Many services integrate with Git for easy updates.

2. Static Site Hosting

Platforms like Vercel and Netlify allow developers to push updates to a Git repository, automatically deploying the latest version of a website.

3. CMS-Based Hosting

For websites built with WordPress, Wix, or Squarespace, hosting and deployment are integrated within the platform, eliminating the need for manual file transfers.

What is Git?

Git is a distributed version control system (VCS) that helps developers track changes to files and collaborate on projects. It was created by Linus Torvalds in 2005 and is widely used in software development, including web development. Git allows multiple people to work on the same project simultaneously without overwriting each other's work, and it keeps a history of all changes made to the project over time.

Organizing Your Website

A well-organized website is essential for easy navigation, maintainability, and scalability. Here are some key principles for structuring your website files effectively:

1. Use a Clear Folder Structure

  • Store HTML files in the root directory (e.g., index.html).

  • Keep styles in a css/ folder.

  • Store JavaScript files in a js/ folder.

  • Place images and media files in an images/ folder.

2. Follow Naming Conventions

  • Use lowercase letters and hyphens instead of spaces (e.g., about-us.html).

  • Keep file and folder names descriptive and meaningful.

3. Use External Stylesheets and Scripts

  • Avoid inline CSS and JavaScript; instead, link to external files.

  • This improves readability and makes maintenance easier.

4. Organize Content Logically

  • Use a consistent navigation structure across pages.

  • Group related files together to improve clarity.

By following these practices, you ensure that your website is structured efficiently, making it easier to update and expand over time.

Introduction to GitHub Pages for Free Hosting

GitHub Pages is a free service offered by GitHub that allows you to host static websites directly from a GitHub repository. It is particularly useful for personal projects, portfolios, documentation, and blogs. The service supports HTML, CSS, and JavaScript, and can also integrate with Jekyll, a static site generator, to simplify the process of creating and maintaining a website.

Step-by-Step Guide to Publishing a Web Page

Step 1: Create a GitHub Account

Before you can publish a web page, you need a GitHub account. If you don't have one, go to GitHub's website and sign up for a free account. Follow the instructions to complete the registration process.

Step 2: Create a New Repository

Once you have a GitHub account, you need to create a new repository to host your web page. Follow these steps:

  1. Log in to your GitHub account.
  2. Click on the "+" icon in the upper-right corner and select "New repository."
  3. Enter a repository name. For a personal website, it is recommended to name it <your-username>.github.io.
  4. Add a description if desired.
  5. Choose the repository visibility: public or private. Public repositories are accessible to everyone, while private ones are restricted to you and collaborators.
  6. Initialize the repository with a README file if you wish.
  7. Click "Create repository."

Step 3: Add Your Web Page Files

With your repository created, you need to add the files that make up your web page. You can do this in several ways:

  • Directly on GitHub: Navigate to your repository, click "Add file," and select "Upload files." Drag and drop your HTML, CSS, and JavaScript files into the upload area.

Step 4: Enable GitHub Pages

To publish your web page, you need to enable GitHub Pages for your repository:

  1. Go to your repository on GitHub.
  2. Click on "Settings."
  3. Scroll down to the "Pages" section.
  4. Under "Source," select the branch you want to use for GitHub Pages, typically "main" or "master."
  5. If your files are in a specific folder, select it under the "Folder" dropdown.
  6. Click "Save."

GitHub will now build your site and provide a URL where it can be accessed, typically https://<your-username>.github.io/<repository-name>/.

Step 5: Customize Your Web Page

Once your web page is live, you can customize it further:

  • Themes: If you are using Jekyll, you can apply themes to your site. Navigate to the "Settings" tab, scroll to the "Pages" section, and select a theme from the "Theme Chooser."
  • Custom Domain: You can use a custom domain by adding a CNAME file to your repository with your domain name and configuring your DNS settings to point to GitHub's servers.

Your Turn: Publish a Simple Website Using GitHub Pages

Follow these steps to complete your hands-on learning task.

Step 1: Create and Upload Your Website

  1. Open VS Code and create a file named index.html and insert the following:

<!DOCTYPE html>
<html>
    <head>
    <title>My Github Web Page</title>
    </head>
<body>
    <h1>Hello!</h1>
    <p>Welcome to my website.</p>
</body>
</html>

  1. Save the file and upload it to your GitHub repository following the earlier steps.

Step 2: Verify Your Website

  1. Wait a few minutes for GitHub Pages to process the deployment.

  2. Open a browser and visit https://yourusername.github.io/my-website/.

  3. Ensure your webpage displays correctly.
