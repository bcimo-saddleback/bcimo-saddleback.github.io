Chapter One

Hosting Options & Installing WordPress

Understand the hosting landscape and get WordPress running on your own server.

What is WordPress?

WordPress is the world's most popular Content Management System (CMS)A CMS is software that allows you to create, manage, and publish digital content without requiring advanced technical knowledge., powering over 43% of all websites on the internet. It began as a blogging platform in 2003 and has evolved into a flexible full-featured web publishing platform.

There are two distinct flavors you should know:

WordPress.com is a hosted platform — Automattic hosts everything for you.

  • ✅ No server management required
  • ✅ Free plan available
  • ⚠️ Limited plugin access on lower tiers
  • ⚠️ Less customization control
  • Best for: personal blogs, simple sites, beginners

WordPress.org is self-hosted — you download the software and run it on your own hosting.

  • ✅ Full control over everything
  • ✅ Install any theme or plugin
  • ⚠️ Requires web hosting & domain
  • ⚠️ You manage backups and updates
  • Best for: businesses, developers, serious bloggers

💡
Recommended for This Course

For learning purposes, use LocalWP (free) to install WordPress on your local machine, or sign up for a free account at WordPress.com. Both give you a sandbox to practice without worrying about server costs.

Installing WordPress (Step-by-Step)

The official motto is "Famous 5-Minute Install." Here's what you need:

1
Purchase Hosting & Domain

Choose a hosting provider (Bluehost, SiteGround, DreamHost). Register a domain name (yoursitename.com).

2
Create a MySQL Database

In your hosting control panel (cPanel), create a new database, username, and password. Note these down.

3
Download & Upload WordPress

Download from wordpress.org/download, then upload via FTP or your host's file manager to the public_html folder.

4
Run the Installer

Navigate to your domain in a browser. The WordPress setup wizard will guide you through connecting to your database.

5
Set Up Your Admin Account

Create a strong username, password, and enter your site title and email. Click "Install WordPress."

The wp-config.php File

During installation, WordPress creates a critical configuration file. Understanding it is essential:

wp-config.php
// Database configuration
define( 'DB_NAME', 'your_database_name' );
define( 'DB_USER', 'your_db_username' );
define( 'DB_PASSWORD', 'your_db_password' );
define( 'DB_HOST', 'localhost' );

// Security Keys (generated automatically)
define( 'AUTH_KEY', 'put your unique phrase here' );
define( 'SECURE_AUTH_KEY', 'put your unique phrase here' );

Chapter Two

Creating & Managing Users

Learn how WordPress handles user accounts, roles, and permissions.

WordPress User Roles

WordPress has a built-in role-based access controlA system where access rights are grouped by role, and users are assigned to roles, making permission management easier. system. Each user is assigned a role that determines what they can and cannot do on your site.

The Administrator has complete control over the WordPress site. They can install/delete plugins and themes, create and manage all users, access all settings, and delete the site entirely.

⚠️ Only trusted individuals should have Admin access. This is your account as the site owner.

Editors can publish, manage, and delete any posts and pages — including those written by other users. They can also moderate comments and manage categories/tags. Great for managing a content team.

Authors can write, publish, edit, and delete their own posts. They can upload files. However, they cannot modify others' content or access site settings. Perfect for trusted writers.

Contributors can write and edit their own posts but cannot publish them — posts must be reviewed and published by an Editor or Admin. They also cannot upload files. Use for guest writers.

Subscribers can only manage their own profile. They cannot create or edit content. Used for membership sites where registered users get access to special content.

Adding a New User

1
Navigate to Users → Add New

In your WordPress admin dashboard (wp-admin), find the Users menu in the left sidebar and click "Add New User."

2
Fill in User Details

Enter username (required), email (required), first/last name, and website URL. The username cannot be changed later.

3
Set a Strong Password

WordPress will generate a strong password. You can use this or create your own. Check "Send User Notification" to email them their credentials.

4
Assign the Appropriate Role

Select the role that matches what this person needs to do. Remember: follow the principle of least privilege — give only the access needed.

Security Best Practice

Never use "admin" as your username — it's the first thing attackers try. Always use a unique username and a strong password with uppercase, lowercase, numbers, and symbols.

Chapter Three

Content Creation & Management

Master the WordPress Block Editor and manage your media library.

The Block Editor (Gutenberg)

Since WordPress 5.0, the default editor is the Gutenberg Block EditorA visual page builder built into WordPress where content is created using individual "blocks" — each block is a discrete piece of content like a paragraph, image, or button.. Everything you create is made of blocks.

ℹ️
What is a Block?

A block is a self-contained content element. Common blocks include: Paragraph, Heading, Image, Gallery, Video, Quote, List, Button, Columns, and hundreds more via plugins.

Essential Block Types

Paragraph — Basic text. Heading (H1–H6) — Section titles. List — Bullet or numbered. Quote — Styled blockquotes. Code — Monospace formatted code. Table — Data tables with rows and columns.

Image — Single photos with alt text, captions, and links. Gallery — Multiple images in a grid. Video — Self-hosted or YouTube embeds. Audio — MP3 and other audio files. Cover — Image with text overlay.

Columns — Side-by-side content. Group — Wrap blocks together with shared styling. Spacer — Add whitespace. Separator — Horizontal dividers. Row — Horizontal block layouts.

WordPress auto-embeds content from YouTube, Twitter/X, Instagram, Spotify, TikTok, and many more. Just paste the URL on its own line, or use the dedicated Embed block.

Managing the Media Library

The Media Library (found under Media in the left sidebar) is where all uploaded files live — images, PDFs, videos, audio, and documents.

1
Uploading Media

Go to Media → Add New. Drag and drop files or click "Select Files." WordPress accepts JPEG, PNG, GIF, WebP, MP4, MP3, PDF, and more.

2
Adding Alt Text

Always add descriptive alt text to images for accessibility and SEO. Click any image in the library to edit its details.

3
Organizing with Folders (Plugin Required)

By default WordPress doesn't have media folders, but plugins like Filester or Enhanced Media Library add this functionality.

Chapter Four

Themes, Widgets & Plugins

Customize the look of your site and extend its functionality.

WordPress Themes

A themeA WordPress theme controls the visual presentation of your website — colors, fonts, layouts, and page structure — without affecting your content. controls the visual design of your website. WordPress has two modern theme types:

Classic Themes use PHP template files, a traditional Customizer, and widget areas.

  • Customized via Appearance → Customize
  • Use widget areas (sidebar, footer)
  • Thousands available at WordPress.org
  • Examples: Astra, OceanWP, GeneratePress

Block Themes (Full Site Editing) allow you to edit every part of your site with the block editor.

  • Edit headers, footers, and templates visually
  • Customized via Appearance → Editor
  • Use Global Styles for site-wide design tokens
  • Examples: Twenty Twenty-Four, Kadence, Blocksy

Installing a Theme

1
Go to Appearance → Themes

In your admin dashboard, navigate to Appearance → Themes to see installed themes.

2
Click "Add New Theme"

Browse thousands of free themes from the official WordPress.org repository. Filter by Popular, Latest, or use the Feature Filter.

3
Preview, Install & Activate

Hover over a theme and click "Preview" to see it live, then "Install" to download it. Click "Activate" to make it your active theme.

Widgets

WidgetsWidgets are small, self-contained modules that add specific functionality to widget-ready areas of your WordPress site, like sidebars and footers. are modular content blocks placed in widget-ready areas (sidebars, footers, headers). In classic themes, go to Appearance → Widgets.

Common widgets include: Recent Posts, Search Bar, Categories, Archives, Tag Cloud, Calendar, Custom HTML, and Text.

ℹ️
Widgets in Block Themes

Block themes don't use traditional widgets. Instead, you add blocks directly to your template parts (header, footer, sidebar) using the Site Editor. All blocks function as "widgets" in this context.

Plugins

PluginsPlugins are packages of code that extend WordPress functionality. There are over 59,000 free plugins in the official WordPress plugin directory. add features to WordPress without editing any code. They range from contact forms to e-commerce to SEO tools.

Chapter Five

Feeds, Podcasting & Social Media Integration

Distribute your content through RSS, podcasts, and social platforms.

RSS Feeds in WordPress

RSS (Really Simple Syndication)An XML-based format that allows users and applications to subscribe to website content updates automatically. Your WordPress site creates RSS feeds automatically. feeds allow readers to subscribe to your content using feed readers like Feedly.

WordPress automatically generates several RSS feeds:

  • Main Feed: yoursite.com/feed/ — all posts
  • Category Feed: yoursite.com/category/name/feed/
  • Comments Feed: yoursite.com/comments/feed/
  • Author Feed: yoursite.com/author/username/feed/
💡
Configure Feed Settings

Go to Settings → Reading to control how many posts appear in your feed and whether to show full content or just a summary excerpt.

Setting Up a Podcast

WordPress can serve as a podcast publishing platform. You need a plugin to add the necessary features:

1
Install a Podcast Plugin

Recommended: Seriously Simple Podcasting or Podlove Podcast Publisher. These add podcast-specific feeds and player.

2
Upload Your Audio Files

Upload MP3 files to your Media Library or use a podcast media host (Buzzsprout, Libsyn, Anchor) for better bandwidth management.

3
Configure Podcast Feed Details

Add show artwork (3000×3000px JPEG), show title, description, category, and author. These are required by Apple Podcasts and Spotify.

4
Submit to Directories

Submit your podcast RSS feed URL to Apple Podcasts, Spotify, Google Podcasts, and Stitcher. They pull new episodes automatically.

Social Media Integration

Integrate WordPress with social platforms in several ways:

Add share buttons to your posts so readers can share to Facebook, Twitter/X, LinkedIn, Pinterest, etc. Popular plugins: Social Snap, AddToAny Share Buttons, Shareaholic.

Automatically share new posts to your social profiles using plugins like Jetpack Social or Blog2Social. Set it up once and every new post gets published to your linked accounts.

Use the Embed block to paste a social media URL and WordPress will display it inline. Works with YouTube, Twitter/X, Instagram (with plugin), TikTok, and more.

Open Graph tags control how your content looks when shared on social media — the title, description, and image shown in the preview card. Yoast SEO and Rank Math automatically add Open Graph tags.

Using & Formatting Posts and Pages

Understand the core content types and master advanced formatting.

Posts vs. Pages

WordPress has two primary content types. Understanding the difference is fundamental:

FeaturePostsPages
PurposeDynamic, time-based content (blogs, news)Static, evergreen content (About, Contact)
Dated✓ Yes✗ No date displayed
Categories & Tags✓ Yes✗ No (by default)
Appears in RSS Feed✓ Yes✗ No
Author Displayed✓ Usually yes✗ Usually no
Hierarchical✗ No✓ Yes (parent/child pages)
Custom TemplatesLimited✓ Full template support

Formatting Posts & Pages

Block Editor Formatting Tips

  • Heading hierarchy — Use H2 for main sections, H3 for sub-sections. Never skip levels.
  • Featured Images — Set a featured image for every post; it appears in listings and social shares.
  • Excerpt — Write a custom excerpt (150–160 characters) for better SEO and RSS display.
  • Columns block — Create magazine-style layouts with the Columns block.
  • Pull quotes — Use the Pull Quote block to highlight key statements.

Adding Images to Content

1
Insert the Image Block

Click the + icon, search for "Image," and insert it. Upload a new image or select from the Media Library.

2
Set Image Size & Alignment

In the right sidebar, choose from Thumbnail, Medium, Large, or Full Size. Set alignment to left, right, center, or wide/full-width.

3
Add Alt Text & Caption

Required for accessibility: add descriptive alt text. Add an optional caption below the image for context.

Post Formats

Some themes support Post Formats — a way to style posts differently based on their content type. Common post formats include:

  • Standard — Default format
  • Image — Photo-focused posts
  • Video — Video-centric posts
  • Quote — Short quotation posts
  • Link — Posts that link to external content
  • Gallery — Image gallery posts
  • Audio — Music or podcast posts

Chapter Seven

Scheduling, Categories, Tags & Comments

Organize content and manage community interactions.

Scheduling Posts

WordPress lets you write content now and publish it automatically at a future date and time. This is essential for content calendars and consistent publishing.

1
Write Your Post

Create and format your post in the Block Editor. Don't click Publish yet.

2
Open the Publish Panel

In the right sidebar under "Post," click "Publish" → "Immediately." A date/time picker will appear.

3
Set Your Publish Date & Time

Choose the date and time you want the post to go live. WordPress uses your site's configured timezone (Settings → General).

4
Click "Schedule"

The button changes from "Publish" to "Schedule." Click it and your post is queued for automatic publication.

Categories vs. Tags

Both are taxonomyA taxonomy is a classification system. WordPress uses taxonomies to group and organize posts. systems for organizing posts, but they serve different purposes:

AspectCategoriesTags
PurposeBroad topic groupingSpecific topic keywords
Hierarchy✓ Parent/child categories✗ Flat, no hierarchy
Required?✓ Yes (defaults to "Uncategorized")✗ Optional
Quantity per post1–3 (ideally)3–10 (ideally)
ExampleTechnology → Web Design#css, #frontend, #responsive
💡
Taxonomy Best Practices

Plan your category structure before publishing. Good categories are like chapters in a book — broad enough to contain multiple posts, specific enough to be meaningful. Don't create a new category unless you plan to have at least 5–10 posts in it.

Managing Comments

Comments allow readers to engage with your content. WordPress gives you full control over the commenting system.

Go to Settings → Discussion to configure: require email/name, hold comments for moderation, auto-approve returning commenters, enable/disable threaded (nested) replies, and set how many comments appear per page.

Go to Comments in the admin sidebar. You can Approve, Spam, Trash, Edit, or Reply to any comment. You can also bulk-approve or bulk-delete selected comments.

Install Akismet Anti-Spam (free for personal sites) — it automatically filters spam comments. Also consider requiring user registration before commenting, or adding a CAPTCHA with WPForms or hCaptcha.

In Settings → Discussion, you can add words, IPs, emails, or URLs to the Comment Moderation list (holds for approval) or Comment Disallowed Keys list (sends directly to trash).

Chapter Eight

WordPress Settings, Security & Maintenance

Configure your site for performance, protect it from threats, and maintain it long-term.

Essential WordPress Settings

After installation, these settings panels should be reviewed immediately:

  • Site Title & Tagline — Used in your browser tab and SEO
  • WordPress Address / Site Address — Your domain URLs (rarely need changing)
  • Admin Email — Where WordPress notifications are sent
  • Timezone — Critical for correct scheduled post publishing
  • Date & Time Format — How dates display on your site
  • Your homepage displays — Choose between latest posts (blog) or a static page (website)
  • Blog pages show at most — Number of posts per page (10 default)
  • For each article in a feed, include — Full text or summary in RSS
  • Search Engine Visibility — Discourage search engine indexing (useful during development)
  • Default post category — What category is pre-selected when creating posts
  • Default post format — Standard, Image, Video, etc.
  • Post via email — Publish posts by sending an email (legacy feature)
  • Update Services — Ping services like Ping-O-Matic when you publish

Permalinks control your URL structure. Always set this immediately after installation!

  • Plain — /?p=123 — Bad for SEO
  • Day and name — /2024/01/01/post-name/
  • Month and name — /2024/01/post-name/
  • Post name ✅ — /post-name/ — Best for SEO!
  • Custom structure — Define your own pattern
  • Thumbnail size — Default: 150×150px
  • Medium size — Default: 300×300px max
  • Large size — Default: 1024×1024px max
  • WordPress creates multiple sizes automatically when you upload an image
  • Enable Organize uploads into month- and year-based folders for cleaner media library

WordPress Security

Security is non-negotiable. WordPress sites are frequent targets because of their popularity. Follow this security checklist:

Use a unique admin username (never "admin"). Use a password with 20+ characters including symbols. Enable Two-Factor Authentication (2FA) with a plugin like WP 2FA or Google Authenticator. Use a password manager.

Update WordPress core, themes, and plugins regularly. 80%+ of WordPress hacks exploit outdated software. Enable automatic minor updates in Settings → Updates or via wp-config.php. Test major updates on a staging site first.

Install Wordfence Security (free) or Sucuri Security. These provide: a web application firewall (WAF), malware scanning, login attempt limiting (brute force protection), and email alerts for suspicious activity.

Your site should use HTTPS (the padlock in the browser). Most hosts provide free SSL via Let's Encrypt. After enabling SSL, update your WordPress Address to https:// in Settings → General. Use the Really Simple SSL plugin to fix mixed content issues.

A backup is your safety net. Use UpdraftPlus (free) to automatically backup your files and database to Google Drive, Dropbox, or Amazon S3. Schedule daily database backups and weekly full-site backups. Test your restores!

By default, WordPress allows unlimited login attempts, making brute-force attacks easy. Use Limit Login Attempts Reloaded or Wordfence to limit failed login tries and temporarily block IPs after X failures.

wp-config.php — Security Additions
// Disable file editing in the dashboard
define( 'DISALLOW_FILE_EDIT', true );

// Force SSL for admin and logins
define( 'FORCE_SSL_ADMIN', true );

// Limit post revisions to save database space
define( 'WP_POST_REVISIONS', 3 );

// Enable automatic core updates
define( 'WP_AUTO_UPDATE_CORE', true );

Site Maintenance Routine

📅
Daily

Check for new comments to moderate. Monitor for security alerts from your security plugin. Review site analytics.

📅
Weekly

Update WordPress core, themes, and plugins. Check that backups are completing successfully. Review uptime monitoring.

📅
Monthly

Optimize the database (WP-Optimize plugin). Delete unused themes and plugins. Test form submissions. Audit user accounts and remove inactive ones.

📅
Quarterly

Test your backup restore process. Review and update outdated content. Check all external links for broken URLs (Broken Link Checker plugin). Review page speed scores.
