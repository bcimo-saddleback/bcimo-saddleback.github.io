Chapter Eight

WordPress Settings, Security & Maintenance

Configure your site for performance, protect it from threats, and maintain it long-term.

Essential WordPress Settings

After installation, these settings panels should be reviewed immediately:

  • Site Title & Tagline — Used in your browser tab and SEO
  • WordPress Address / Site Address — Your domain URLs (rarely need changing)
  • Admin Email — Where WordPress notifications are sent
  • Timezone — Critical for correct scheduled post publishing
  • Date & Time Format — How dates display on your site
  • Your homepage displays — Choose between latest posts (blog) or a static page (website)
  • Blog pages show at most — Number of posts per page (10 default)
  • For each article in a feed, include — Full text or summary in RSS
  • Search Engine Visibility — Discourage search engine indexing (useful during development)
  • Default post category — What category is pre-selected when creating posts
  • Default post format — Standard, Image, Video, etc.
  • Post via email — Publish posts by sending an email (legacy feature)
  • Update Services — Ping services like Ping-O-Matic when you publish

Permalinks control your URL structure. Always set this immediately after installation!

  • Plain — /?p=123 — Bad for SEO
  • Day and name — /2024/01/01/post-name/
  • Month and name — /2024/01/post-name/
  • Post name ✅ — /post-name/ — Best for SEO!
  • Custom structure — Define your own pattern
  • Thumbnail size — Default: 150×150px
  • Medium size — Default: 300×300px max
  • Large size — Default: 1024×1024px max
  • WordPress creates multiple sizes automatically when you upload an image
  • Enable Organize uploads into month- and year-based folders for cleaner media library

WordPress Security

Security is non-negotiable. WordPress sites are frequent targets because of their popularity. Follow this security checklist:

Use a unique admin username (never "admin"). Use a password with 20+ characters including symbols. Enable Two-Factor Authentication (2FA) with a plugin like WP 2FA or Google Authenticator. Use a password manager.

Update WordPress core, themes, and plugins regularly. 80%+ of WordPress hacks exploit outdated software. Enable automatic minor updates in Settings → Updates or via wp-config.php. Test major updates on a staging site first.

Install Wordfence Security (free) or Sucuri Security. These provide: a web application firewall (WAF), malware scanning, login attempt limiting (brute force protection), and email alerts for suspicious activity.

Your site should use HTTPS (the padlock in the browser). Most hosts provide free SSL via Let's Encrypt. After enabling SSL, update your WordPress Address to https:// in Settings → General. Use the Really Simple SSL plugin to fix mixed content issues.

A backup is your safety net. Use UpdraftPlus (free) to automatically backup your files and database to Google Drive, Dropbox, or Amazon S3. Schedule daily database backups and weekly full-site backups. Test your restores!

By default, WordPress allows unlimited login attempts, making brute-force attacks easy. Use Limit Login Attempts Reloaded or Wordfence to limit failed login tries and temporarily block IPs after X failures.

wp-config.php — Security Additions
// Disable file editing in the dashboard
define( 'DISALLOW_FILE_EDIT', true );

// Force SSL for admin and logins
define( 'FORCE_SSL_ADMIN', true );

// Limit post revisions to save database space
define( 'WP_POST_REVISIONS', 3 );

// Enable automatic core updates
define( 'WP_AUTO_UPDATE_CORE', true );

Site Maintenance Routine

📅
Daily

Check for new comments to moderate. Monitor for security alerts from your security plugin. Review site analytics.

📅
Weekly

Update WordPress core, themes, and plugins. Check that backups are completing successfully. Review uptime monitoring.

📅
Monthly

Optimize the database (WP-Optimize plugin). Delete unused themes and plugins. Test form submissions. Audit user accounts and remove inactive ones.

📅
Quarterly

Test your backup restore process. Review and update outdated content. Check all external links for broken URLs (Broken Link Checker plugin). Review page speed scores.
