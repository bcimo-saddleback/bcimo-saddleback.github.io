Chapter6: Web Design Basics

Introduction

Web design involves creating visually appealing and functional web pages. A fundamental part of web design includes structuring elements, styling them, and ensuring a cohesive layout. In this chapter, you will gain a basic understanding of CSS (Cascading Style Sheets).

Understanding HTML and CSS

HTML (HyperText Markup Language) is used to structure web pages, while CSS (Cascading Style Sheets) is used to style them. CSS allows for adding colors, spacing, and positioning elements to create an engaging user experience.

Basic CSS Syntax

CSS consists of selectors and declarations. Here's an example:

h1 {
 color: blue; /* Property: color, Value: blue */
 font-size: 24px; /* Property: font-size, Value: 24px */
}
  • Selector: h1
    • This targets all <h1> elements on the page.
  • Declarations:
    • color: blue; — This changes the text color of <h1> elements to blue.
    • font-size: 24px; — This changes the font size of <h1> elements to 24 pixels.

Summary:

  • The selector identifies which HTML element to style.
  • The declaration defines what style to apply, with each declaration made up of a property and its value.

Adding CSS to HTML

To add CSS directly to an HTML document, you can use the <style> tag within the <head> section of the document. Here's how you do it:

You need to place the <style> tag inside the <head> element of your HTML document. Anything you write inside the <style> tag will be applied as CSS styles to the HTML content on that page.

Example:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page</title>

    <style>
        /* CSS styles go here */
        h1 {
            color: blue;
            font-size: 24px;
        }

        p {
            color: green;
            font-size: 16px;
        }
    </style>
</head>
<body>
    <h1>Welcome to My Page</h1>
    <p>This is a paragraph of text.</p>
</body>
</html>

Key Points:

  1. <style> tag:

    • It's used to contain CSS code.
    • It's placed inside the <head> section of the HTML document.
    • CSS code inside the <style> tag applies to the entire HTML document.

  2. Inside the <style> tag:

    • You write your regular CSS rules (selectors and declarations) like you normally would in an external stylesheet.

Why use the <style> tag?

  • It’s a good way to include CSS when working on a small project or when you want to quickly test styles.
  • For larger projects or when you want to keep the CSS separate, it’s better to use an external CSS file linked with the <link> tag.

The above code will apply the following styles:

  • The <h1> will be blue and have a font size of 24px.
  • The <p> will be green and have a font size of 16px.

External CSS

External CSS involves linking an HTML document to an external .css file. This method is the most efficient for larger projects, as it allows for a clean separation of content and design. It also makes it easier to maintain and update styles across multiple web pages by editing a single CSS file.

To use external CSS, you create a separate CSS file (e.g., styles.css) and link it to your HTML document using the <link> tag in the <head> section.

Example of an HTML file linking to an external CSS file (try this in the code runner below):

<head>
     <meta http-equiv=Content-Type content="text/html; charset=utf-8">
     <title>Web Design Basics</title>
     <link rel="stylesheet" href="style.css">
 </head>
 <body>
     <header>
         <h1>Welcome to Web Design Basics</h1>
     </header>

    <div class="summary" id="zen-summary" role="article">
        <p>A demonstration of what can be accomplished through <abbr title="Cascading Style Sheets">CSS</abbr>-based design.
    </div>
    <h3>The Road to Enlightenment</h3>
<p>A demonstration of what can be accomplished through CSS-based design.</p>
</body>
</html>

Advantages of Using External CSS

  • Consistency: External CSS allows for consistent styling across multiple web pages. By linking all pages to a single CSS file, you ensure that changes to the design are applied universally.

  • Efficiency: External CSS reduces redundancy and file size in HTML documents, as styles are defined once in a separate file rather than repeated within each HTML document.

Introduction to CSS Zen Garden

CSS Zen Garden is an online resource that allows designers to submit their own CSS designs to transform a single HTML file. The purpose of this site is to highlight the capabilities of CSS in creating visually appealing web pages without altering the HTML structure. By visiting CSS Zen Garden, you can explore a variety of designs, each demonstrating unique styles and layouts, all applied to the same HTML content.

Conclusion

In this chapter, we’ve explored the fundamentals of web design, focusing on the essential roles of HTML and CSS. HTML is the backbone of web structure, allowing us to define the content and layout of web pages. CSS, on the other hand, is the tool that allows us to style and enhance these elements, ensuring that they are visually appealing and user-friendly.

We’ve learned how to structure and apply CSS in different ways, whether through internal styles within a <style> tag or by linking to external CSS files. Each method offers its own benefits, with external CSS being ideal for larger projects due to its ability to maintain consistency and efficiency across multiple pages.

The chapter also introduced the concept of CSS Zen Garden, demonstrating the power of CSS in transforming the visual presentation of the same HTML structure. This exemplifies the core principle of web design: separating content (HTML) from presentation (CSS).

By understanding and applying the basic principles and techniques covered in this chapter, you are now equipped to create well-structured, stylish, and functional web pages. As you continue to experiment with CSS, you’ll be able to refine your skills and explore more advanced design techniques to enhance your web design projects.
