Section 1: Inserting Images Using the <img> Tag

In HTML, images are an essential part of web content, making sites more engaging and visually appealing. One of the most fundamental methods for inserting images into a webpage is by using the <img> tag. This tag allows you to display various types of images, including photographs, illustrations, icons, and logos, directly in your HTML document. Let’s explore how you can use this tag and understand its key attributes, as well as how to incorporate images efficiently into your web pages.

Basic Syntax

The <img> tag itself is an empty or self-closing tag, which means it doesn't require an end tag. The basic syntax for using the <img> tag is:

<img src="image-source.jpg" alt="description of image">
  • src: The src (source) attribute is used to specify the path to the image you want to display. This path can either be relative (pointing to a location on your server) or absolute (an external URL).
  • alt: The alt (alternative text) attribute is a crucial part of the <img> It provides a description of the image in case the image is not loaded or for users with visual impairments who use screen readers. For example, if you were displaying a photo of a dog, your alt text might be “A cute dog playing in the park.”

Here’s a simple example:

<img src="dog.jpg" alt="A cute dog playing in the park"> 

Sizing Images

Sometimes images may need resizing to fit into your webpage’s layout. While you can resize an image using CSS or HTML attributes, it’s always best practice to ensure images are optimized for both appearance and performance. You can specify the width and height of an image directly within the HTML tag: 

<img src="cat.jpg" alt="A playful cat" width="300" height="200">

This will display the image with a width of 300 pixels and a height of 200 pixels. Be careful not to distort the image by forcing a non-proportional size.

Image Formats

Common image formats supported by web browsers include:

  • JPEG/JPG: Best for photographs, with high compression and small file sizes.
  • PNG: Supports transparency and is great for graphics like logos and icons.
  • GIF: Best for animations and simple graphics.
  • SVG: Scalable vector graphics that are resolution-independent and ideal for logos and icons.

To insert an image, simply use the correct file format in the src attribute:

<img src="logo.svg" alt="Company Logo"> 

The title Attribute

The title attribute can be added to the <img> tag to provide additional information about the image. This will appear when the user hovers over the image. Although it's not essential for functionality, it can improve the user experience.

<img src="images/photo.jpg" alt="A beautiful sunset over the ocean" title="Sunset at the beach">

This will show a tooltip with the text "Sunset at the beach" when the user hovers over the image.

Best Practices for Images

When inserting images into your webpage, remember to:

  • Use descriptive alt text for accessibility and SEO.
  • Optimize image file sizes to reduce loading times.
  • Use responsive design principles, ensuring images adapt to various screen sizes.

By mastering the use of the <img> tag and incorporating these best practices, you’ll enhance the user experience of your website while ensuring your content is both visually engaging and accessible.

 

Section 2: Introduction to Multimedia: Embedding Videos and Audio

While images enhance the visual appeal of a website, videos and audio are essential for creating more interactive and engaging experiences. HTML provides simple tags to embed video and audio files directly into a webpage.

Embedding Videos with the <video> Tag

The <video> tag allows you to embed video files. You can specify multiple video sources for better compatibility across browsers, and add controls to allow users to play, pause, or adjust the volume.

<video src="video.mp4" controls>  Your browser does not support the video tag.</video>

In this example, the video file will be displayed. The controls attribute adds the standard video controls, such as play, pause, and volume.

  • src: This attribute specifies the video file you want to embed. You can use relative paths (if the video is hosted on the same server) or absolute URLs (if the video is hosted externally).
  • controls: This optional attribute adds built-in controls such as play, pause, volume, and full-screen options to the video player.
  • Fallback Text: The content inside the <video> tag (like "Your browser does not support the video tag") is displayed if the user's browser doesn’t support the video element.

Section 3: Image File Types and When to Use Them

Choosing the right image file type is important for both performance and image quality. Let’s explore the most common image file formats used in web development.

JPEG (Joint Photographic Experts Group)

JPEG is the most widely used image format for photographs and complex images. It supports millions of colors and provides high compression rates, making it ideal for images with subtle color variations.

  • When to use: Photographs, images with gradients, or detailed artwork.
  • Pros: Smaller file sizes with high-quality images.
  • Cons: Lossy compression (some quality is lost when saving).

PNG (Portable Network Graphics)

PNG is a lossless image format, meaning no data is lost during compression. It supports transparency, which is useful for images that need to blend seamlessly with the background.

  • When to use: Logos, icons, images with transparency, or sharp-edged graphics.
  • Pros: Lossless quality, supports transparency.
  • Cons: Larger file sizes compared to JPEG.

GIF (Graphics Interchange Format)

GIF is often used for simple animations and images with few colors. It supports transparency but only allows one color to be fully transparent.

  • When to use: Simple animations, low-resolution images, or pixel art.
  • Pros: Supports animation, supports transparency.
  • Cons: Limited to 256 colors, not suitable for high-quality images.

SVG (Scalable Vector Graphics)

SVG is a vector-based image format, meaning it uses mathematical equations to draw images. This makes SVG images infinitely scalable without losing quality.

  • When to use: Logos, icons, and illustrations that need to scale across devices.
  • Pros: Scalable, smaller file sizes for simple graphics, and supports animation and interactivity.
  • Cons: Not suitable for complex images like photographs.
