Chapter 2: Introduction to HTML and Web Page Structure

Overview

In this chapter, you will learn the basics of HTML, the foundational language used to create web pages. By the end of this chapter, you will be able to create a simple HTML document and understand its basic structure and essential elements.

What is HTML and Why it's important

HTML stands for HyperText Markup Language. It is the standard language used to create and structure web pages and web applications. HTML provides the building blocks for web content, defining elements such as headings, paragraphs, images, links, and more.

Why Learn HTML?

  • Universal Foundation: HTML is the backbone of web development, regardless of the tools or frameworks you use.
  • Essential for Communication: Understanding HTML allows developers, designers, and marketers to work together effectively.
  • Control and Customization: Learning HTML gives you more control over the content and structure of your web pages.

HTML Tags

HTML tags are the fundamental building blocks of web pages. They are used to structure content, define elements, and provide instructions to web browsers on how to display information. Each HTML tag consists of a name enclosed in angle brackets, such as <p> for a paragraph or <a> for a hyperlink.

Tags often come in pairs, with an opening tag and a closing tag, like <h1> and </h1>, to denote the start and end of an element. The opening tag, <h1>, signals the start of an element, while the closing tag, </h1>, indicates its end. The key difference lies in the forward slash / present in closing tags, which denotes the conclusion of the element. This pairing ensures that content is properly encapsulated and formatted. Understanding this distinction is crucial for creating well-structured web pages, as it helps browsers accurately interpret and display the intended layout and style.

Basic Structure of an HTML Document

An HTML document has a simple and predictable structure, making it easy to learn and use. Here is a simple example of a webpage in HTML: 

<!DOCTYPE html>
<html>
    <head>
        <title>My Web Page</title>
    </head>
    <body>
        <h1>Welcome to beginning web development!</h1>
        <p>I'm glad you are here.</p>
    </body>
</html>

What do all those tags do?

  1. <!DOCTYPE html>: Declares the document type and version of HTML being used (HTML5).
  2. <html>: The root element of an HTML document, enclosing all content.
  3. <head>: Contains metadata about the document, such as the title and links to stylesheets or scripts.
  4. <title>: Sets the title of the web page, which appears on the browser tab.
  5. <body>: Contains the content that is displayed on the web page, including text, images, and other elements.

Common HTML Tags

1. <html>

The <html> tag is the root element of an HTML document. It signifies the beginning and end of an HTML document and encompasses all other HTML elements. The structure of an HTML document begins with the opening <html> tag and concludes with the closing <html> tag. This tag informs the browser that the content within these tags is HTML code. Here is a simple example of how the <html> tag is used: 

<!DOCTYPE html>\n<html>\n  <!-- Other HTML elements go here -->\n</html>

2. <head>

The <head> tag is a container for metadata and links to external resources. Metadata includes information about the document such as its title, character set, styles, scripts, and other meta-information. The content within the <head> tag is not displayed on the webpage itself but is essential for the proper functioning and identification of the page. Here is an example of how the <head> tag might be used: 

<html>\n  <head>\n    <meta charset="UTF-8">\n    <title>My Web Page</title>\n    <link rel="stylesheet" href="styles.css">\n    <script src="script.js"></script>\n  </head>\n  <!-- Body content goes here -->\n</html>

3. <body>

The <body> tag contains the actual content of the webpage that is visible to users. This includes text, images, links, tables, and any other elements that make up the main part of the webpage. The <body> tag follows the <head> tag and is closed just before the closing </html> tag. Here is an example of how the <body> tag might be used: 

<html>\n  <head>\n    <!-- Head content goes here -->\n  </head>\n  <body>\n    <h1>Welcome to My Web Page</h1>\n    <p>This is a paragraph of text on my webpage.</p>\n  </body>\n</html>

4. <title>

The <title> tag is used to set the title of the webpage, which is displayed on the browser's title bar or tab. It is a crucial part of the <head> section and helps users identify the page they are viewing. The <title> tag should be concise and descriptive to effectively convey the content of the page. Here is an example of how the <title> tag might be used: 

<html>\n  <head>\n    <title>Introduction to HTML Tags</title>\n  </head>\n  <body>\n    <!-- Body content goes here -->\n  </body>\n</html>

5. <div>

The <div> tag, short for "division," is a block-level element used to group together larger sections of HTML content. It is commonly used to create layout structures, such as headers, footers, sidebars, and main content areas. The <div> tag does not inherently carry any specific styling or semantic meaning, but it can be styled using CSS (Cascading Style Sheets) to achieve the desired appearance. For example, a <div> can be used to create a container for a website's main content: 

<div class="main-content">\n    <h1>Welcome to My Website</h1>\n    <p>This is the main section of the website where content is displayed.</p>\n</div>

In this example, the div tag is used to encapsulate a header and a paragraph, allowing developers to apply styles or scripts to this section as a whole.

6. <span>

The <span> tag is an inline element used to apply styles or manipulate specific parts of text within a larger block of content. Unlike <div>, which affects entire sections, <span> is used for smaller portions, such as individual words or phrases. The <span> tag is often used in conjunction with CSS to change the appearance of text. For instance: 

<p>This is an example of a <span style="color: red;">highlighted</span> word.</p>

Here, the span tag is used to change the color of the word "highlighted" to red, demonstrating how span can be used to target specific text for styling.

7. <a> (Anchor)

The <a> tag, known as the anchor tag, is used to create hyperlinks, which are essential for navigating the web. The href attribute within the <a> tag specifies the URL of the page the link points to. Hyperlinks can link to other web pages, files, email addresses, or specific sections within the same page. An example of an anchor tag is: 

<a href="https://www.example.com">Visit Example.com</a>

This code creates a clickable link that directs users to "https://www.example.com". The text "Visit Example.com" is the clickable part of the link, known as the anchor text.

8. <br>

The <br> tag, short for "break," is an empty tag used to insert a line break in the text. It is useful for formatting text where a new line is needed without starting a new paragraph. The <br> tag does not have a closing tag, and it is written as a self-closing tag in HTML5. For example: 

<p>This is the first line.<br>This is the second line.</p>

In this example, the br tag creates a line break between "This is the first line." and "This is the second line.", allowing them to appear on separate lines within the same paragraph.

9. <nav>

The <nav> tag is used to define a set of navigation links. It is a semantic element that helps search engines and screen readers understand the structure of a webpage. Typically, the <nav> tag is used to enclose the primary navigation menu of a website, which may include links to other pages or sections within the same page.

Example of nav tag usage:

<nav>\n  <ul>\n    <li><a href="#home">Home</a></li>\n    <li><a href="#about">About</a></li>\n    <li><a href="#services">Services</a></li>\n    <li><a href="#contact">Contact</a></li>\n  </ul>\n</nav>

In this example, the <nav> tag contains an unordered list (ul) of links (a tags) that serve as the main navigation for a website.

10. <section>

The <section> tag is used to define a thematic grouping of content, typically with a heading. It is a semantic tag that helps organize content into logical sections. Each <section> can be thought of as a distinct part of the document that can stand alone in terms of its content.

Example of <section> tag usage with an ID:

<section id="about">\n  <h2>About Us</h2>\n  <p>We are a company dedicated to providing the best services to our clients.</p>\n</section>

In this example, the <section> tag is used to group content related to the "About Us" part of the webpage. The id attribute is used to uniquely identify this section, which can be useful for linking directly to it from other parts of the webpage or from external sources.

11. <footer>

The <footer> tag is used to define the footer of a document or a section. It typically contains information about the author, copyright information, contact details, or links to related documents. The <footer> tag is a semantic element that helps delineate the end of a section or the entire webpage.

Example of <footer> tag usage:

<footer>\n    Contact Us at: <a href="mailto:bcimo@saddleback.edu">bcimo@saddleback.edu</a>\n</footer>

In this example, the <footer> tag contains copyright information and a contact email, providing users with essential details about the webpage.

12. Heading Tags (<h1> to <h6>)

Heading tags, ranging from <h1> to <h6>, are used to define headings in a document. The <h1> tag is the highest level of heading and is typically used for the main title of a page, while <h6> is the lowest level. Headings help to organize content and make it more accessible to users and search engines.

Example:

<h1>Main Title of the Document</h1>\n<h2>Subheading Level h2</h2>\n<h3>Subheading Level h3</h3>

In this example, the <h1> tag is used for the main title, while <h2> and <h3> are used for subheadings. Using headings appropriately helps create a clear hierarchy of information.

13. The Ethics of Web Development: Accessibility & Digital Inclusion

Writing HTML code is not merely a technical configuration task; it carries profound **ethical responsibilities**. When we develop structures for the web, we act as digital architects building the infrastructure of modern human society.

Accessibility as a Matter of Civil Rights

In the contemporary physical world, structural civil rights guarantees dictate that buildings include access ramps, tactile paving, and audio cues. Because critical functions of human life—applying for employment, submitting tax filings, engaging in higher education, and managing personal health—have transitioned online, **web accessibility is a direct matter of digital inclusion and civil rights**.

When a developer deploys non-semantic, poorly structured code lacking proper indicators, it creates an invisible barrier that completely bars individuals with visual, motor, auditory, or cognitive disabilities from accessing critical data resources.

Key Pillars of Ethical Structure:

  • Universal Digital Inclusion: Constructing applications that guarantee equal access pathways for every single visitor, regardless of what assistive hardware, navigation devices, or browser tools they use.
  • The Curb-Cut Effect: An design ethos proving that features built intentionally to support vulnerable populations inherently improve accessibility conditions for all users (e.g., adding structured semantic blocks helps screen readers navigate, but it also improves search engine indexing efficiency for SEO optimization).
  • Semantic Compliance: Committing to elements like <nav>, <section>, and alt text fields because they serve as foundational anchors for assistive technologies to read and navigate interfaces smoothly.
