Key Spectral Lines in SDSS

Why lines matter: Spectral lines are the anchors of redshift measurement. Each element emits and absorbs photons at fixed rest wavelengths set by quantum mechanics. By measuring how far an observed line has shifted from its lab wavelength, SDSS computes z to four decimal places — placing every galaxy precisely in cosmic time and space.

Emission Lines
Star-Forming & Active Galaxies
Emission lines arise when electrons cascade down to lower energy states, releasing photons of specific energy. [O II] 3727Å and [O III] 5007Å trace ionized gas in H II regions and AGN narrow-line regions. Hα is the primary star formation rate tracer in nearby galaxies. These lines dominate galaxy spectra at z < 0.4 where SDSS's wavelength range covers them.
Absorption Lines
Passive Galaxies & Stellar Atmospheres
Absorption lines form when photons are removed by cooler foreground material. The Ca II H&K doublet (3968/3934Å) dominates passive elliptical galaxy spectra — the single most used pair for low-z redshifts. The Mg b triplet (~5175Å) traces stellar velocity dispersion. The 4000Å break — a flux discontinuity from metal line blanketing — encodes galaxy age.
Signal-to-Noise
S/N and Spectral Resolution R
S/N per pixel determines whether faint lines are detectable above noise. SDSS targets S/N > 4 per pixel at the fiber magnitude limit. Spectral resolution R = λ/Δλ ≈ 2000 for SDSS, meaning lines separated by >3Å are resolved. Equivalent width (EW) measures line strength relative to the continuum regardless of overall flux level.
Redshift Formula
z = (λ_obs − λ_rest) / λ_rest
Cosmological redshift z encodes lookback time. At z = 0.1, the universe was ~1.3 Gyr younger than today. At z = 1, ~8 Gyr younger. For low-z, velocity v ≈ cz; at high z, the proper distance requires integrating the comoving metric. SDSS SpecObj stores both the pipeline redshift z and its uncertainty zErr.

The SDSS Spectrograph System

Plates as data identifiers: Every SDSS spectrum is uniquely identified by three numbers — plate (which aluminum disk), MJD (which night), and fiberID (which of the 640 holes). These three integers together point to exactly one spectrum in the archive, and they appear in every filename, database join, and paper citation.
The Plug Plate
3° Field, 640 Fibers, Drilled by Hand
Each aluminum plate covers a 3°-diameter field. Holes are drilled at the exact sky positions of targets selected from SDSS photometry. Optical fibers (180μm diameter, 3″ on-sky) are plugged by hand at Apache Point each afternoon. BOSS plates accommodate 1,000 fibers with 62″ minimum separation due to fiber collision constraints.
The Spectrograph
Wavelength Coverage: 3800–9200Å
SDSS uses two fiber-fed spectrographs (320 fibers each). A dichroic splits each fiber's light into a blue channel (3800–6165Å) and red channel (5765–9200Å). The logarithmic wavelength grid has constant pixel spacing: Δlog₁₀(λ) = 10⁻⁴, giving ~69 km/s per pixel — ideal for velocity measurements.
FITS File Structure
spPlate HDU Layout
The primary HDU contains a 2D image where each row is one spectrum (640 rows × ~3800 wavelength pixels). HDU2 holds the inverse variance (noise) array; HDU3 holds the and-mask/or-mask bitmasks; HDU4 holds the wavelength solution. The loglam array gives log₁₀(λ) in Å — convert with 10**loglam.
Sky Subtraction
32 Sky Fibers Per Plate
About 32 fibers per plate are intentionally placed on blank sky, measuring the night-sky emission spectrum (bright OH airglow lines in the red). The pipeline median-combines these to form a sky model, which is subtracted from every object fiber. The quality of sky subtraction limits S/N for faint targets in the red.

SpecObj Table: Key Columns

Column Type / Range Meaning & Usage
specObjID bigint (64-bit) Unique spectrum identifier — use for cross-table JOINs; encodes plate/MJD/fiberID/rerun
z float, 0–7 Pipeline redshift — primary distance estimator; always check zWarning before use
zErr float Redshift uncertainty (1σ); typical galaxy value ~3×10⁻⁵; high zErr flags noisy spectra
zWarning bitmask integer Quality flags (see table below); zWarning = 0 means clean redshift — always filter on this
class string 'GALAXY', 'STAR', or 'QSO' — pipeline spectral classification; more reliable than photometric type
subClass string Finer classification: 'AGN', 'BROADLINE', 'STARFORMING', 'STARBURST', 'SEYFERT', etc.
plate integer Plug plate number — first element of the plate/MJD/fiberID triplet identifying the spectrum
mjd integer Modified Julian Date of observation; convert with astropy.time.Time(mjd, format='mjd')
fiberID 1–640 (SDSS) / 1–1000 (BOSS) Which fiber collected this spectrum; with plate+MJD gives the FITS file row number
velDisp km/s Stellar velocity dispersion σ — measured from absorption line broadening; proxy for BH mass
elodieTeff Kelvin (stars) Effective temperature from ELODIE template fit — valid for class = 'STAR' only
sciencePrimary 0 or 1 = 1 flags the best-quality spectrum when multiple exist for same object; always use = 1

zWarning Bitmask — Common Flags

Bit / Value Name Meaning Action
0 — No warnings — clean, reliable redshift use
bit 0 (1) SKY Spectrum is a sky fiber, not an object discard
bit 1 (2) LITTLE_COVERAGE Wavelength coverage too small to fit template discard
bit 2 (4) SMALL_DELTA_CHI2 Best-fit z not significantly better than next-best — redshift ambiguous caution
bit 4 (16) MANY_OUTLIERS Many pixels masked as bad — spectrum quality poor caution
bit 7 (128) NEGATIVE_MODEL Best-fit template is unphysically negative discard

Fetching & Analyzing an SDSS Spectrum

This workflow fetches a galaxy spectrum directly from the SDSS SAS (Science Archive Server), reads its FITS structure, identifies emission lines, and computes the redshift manually to compare against the pipeline value in SpecObj.

Python 3 · Jupyter Notebook week3_sdss_spectra.ipynb 
import numpy as np
import matplotlib.pyplot as plt
from astropy.io import fits
from astropy.table import Table
import urllib.request

# ─── 1. Download a spectrum from the SDSS SAS ─────────────────────────────
# URL pattern for spSpec (individual spectrum FITS files)
plate, mjd, fiber = 0266, 51602, 0001   # first SDSS Legacy plate
url = (f"https://dr17.sdss.org/sas/dr17/sdss/spectro/redux/26/spectra/"
       f"{plate:04d}/spec-{plate:04d}-{mjd}-{fiber:04d}.fits")
urllib.request.urlretrieve(url, "spectrum.fits")

# ─── 2. Read the FITS structure ───────────────────────────────────────────
hdul  = fits.open("spectrum.fits")
# HDU 0: header with z, class, plate/mjd/fiber
z_pipeline = hdul[0].header['Z']
obj_class   = hdul[0].header['CLASS']
z_warning   = hdul[0].header['ZWARNING']
# HDU 1: binary table — flux, loglam, ivar, and_mask
spec   = Table(hdul[1].data)
flux   = spec['flux']          # 10⁻¹⁷ erg/s/cm²/Å
loglam = spec['loglam']        # log₁₀(wavelength in Å)
ivar   = spec['ivar']          # inverse variance
wave   = 10 ** loglam           # wavelength in Å (observed frame)

print(f"Object class: {obj_class} | Pipeline z = {z_pipeline:.5f} | zWarning = {z_warning}")

# ─── 3. Plot the spectrum ─────────────────────────────────────────────────
fig, ax = plt.subplots(figsize=(12, 4))
# Mask bad pixels (ivar == 0)
good = ivar > 0
ax.plot(wave[good], flux[good], color='#7eb3ff', lw=0.6, alpha=0.9)
ax.set_xlabel('Observed wavelength (Å)')
ax.set_ylabel('Flux (10⁻¹⁷ erg/s/cm²/Å)')
ax.set_title(f'SDSS spec-{plate:04d}-{mjd}-{fiber:04d}  [{obj_class}  z={z_pipeline:.4f}]')

# ─── 4. Mark known emission lines redshifted to observed frame ───────────
rest_lines = {
    '[OII] 3727':  3727.0,
    'Hβ':          4861.3,
    '[OIII] 5007': 5006.8,
    'Hα':          6562.8,
    '[NII] 6583':  6583.5,
}
for name, lam_rest in rest_lines.items():
    lam_obs = lam_rest * (1 + z_pipeline)
    ax.axvline(lam_obs, color='#3dd6b5', lw=0.8, alpha=0.7, ls='--')
    ax.text(lam_obs, ax.get_ylim()[1] * 0.92, name, fontsize=7,
           color='#3dd6b5', rotation=90, va='top')
plt.tight_layout()

# ─── 5. Manual redshift from a detected emission line ─────────────────────
# Fit a Gaussian to the Hα line to measure its observed wavelength
from scipy.optimize import curve_fit

lam_ha_rest = 6562.8                         # Hα rest wavelength (Å)
lam_ha_obs  = lam_ha_rest * (1 + z_pipeline) # expected observed wavelength
window      = (wave > lam_ha_obs - 50) & (wave < lam_ha_obs + 50)

def gaussian(x, amp, mu, sigma, continuum):
    return amp * np.exp(-0.5 * ((x - mu) / sigma)**2) + continuum

p0 = [flux[window].max(), lam_ha_obs, 5.0, np.median(flux[window])]
popt, _ = curve_fit(gaussian, wave[window], flux[window], p0=p0)
z_manual = (popt[1] / lam_ha_rest) - 1

print(f"Pipeline z = {z_pipeline:.5f}")
print(f"Manual z   = {z_manual:.5f}  (Δz = {abs(z_manual - z_pipeline):.1e})")

By the end of this notebook you will have fetched real SDSS FITS spectra, identified emission and absorption lines by eye, computed redshifts manually from Gaussian fits, and compared your results to the pipeline — the same cross-check professional astronomers run on every new survey data release.

  1. 01 Query SpecObj via CasJobs for 10 galaxies with zWarning = 0, class = 'GALAXY', and a range of redshifts (0.05 < z < 0.4). Record their plate/MJD/fiberID triplets.
  2. 02 Download each spectrum FITS file from the DR17 SAS using the URL pattern above. Load each spectrum's flux, loglam, and ivar arrays. Convert loglam → wavelength in Å.
  3. 03 Plot all 10 spectra in a grid. For each, overlay vertical dashed lines at the observed positions of [O II] 3727, Hβ, [O III] 5007, Hα, and Ca II H&K, using the pipeline z. Verify the lines visually align with features in the spectrum.
  4. 04 For 3 of your galaxies, fit a Gaussian to the Hα line using scipy.optimize.curve_fit. Compute z_manual from the fitted line center. Calculate Δz = |z_manual − z_pipeline| and Δv = c × Δz in km/s. Discuss the discrepancy.
  5. 05 Use specutils to load one of your spectra as a Spectrum1D object. Apply a rest-frame correction using the pipeline z, then use specutils.analysis.equivalent_width to measure the Hα equivalent width and compare to the BPT diagram classification.
  6. 06 Repeat steps 2–4 for one class = 'QSO' spectrum with z > 1.5. Identify Lyα, C IV 1549Å, and Mg II 2798Å in the SDSS wavelength window. Compute z from each line independently and compare the three values.
